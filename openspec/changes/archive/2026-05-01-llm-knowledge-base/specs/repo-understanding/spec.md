## MODIFIED Requirements

### Requirement: Reflection-driven repo memory updates
Component dossiers and fix-pattern memory SHALL be updatable via promoted reflection candidates only. Each update SHALL record the source reflection candidate ID, reviewer, and promotion timestamp.

#### Scenario: Component dossier updated
- **WHEN** a reflection reveals that a predicted component was incorrect and the actual component is confirmed by a reviewer
- **THEN** the component dossier is updated and the update is recorded with `{reflection_candidate_id, reviewer, promoted_at}`

## ADDED Requirements

### Requirement: GitHub file content fetching
The repo adapter SHALL expose a `getFileContent(owner, repo, path, ref?)` method that fetches the raw content of a single file. The method SHALL enforce a 100 KB size limit per file and SHALL throw a `FileTooLargeError` if exceeded. The method SHALL be retried up to 3 times with exponential back-off on transient failures.

#### Scenario: File content fetched successfully
- **WHEN** `getFileContent("org", "repo", "src/auth/AuthService.ts")` is called
- **THEN** the raw file content is returned as a UTF-8 string

#### Scenario: File too large
- **WHEN** the requested file exceeds 100 KB
- **THEN** `FileTooLargeError` is thrown and no content is returned

### Requirement: LLM-grounded solution plan
After the component mapper selects candidate components and files, the solution planner SHALL call the LLM adapter to produce a `why` justification string for each candidate component and candidate file. The `why` field SHALL reference specific code or KT context. If the LLM is degraded, `why` SHALL be set to the existing heuristic description.

#### Scenario: LLM populates why field
- **WHEN** LLM grounding is available and file content has been fetched
- **THEN** each `CandidateComponent.why` and `CandidateFile.reason` contains a non-empty LLM-generated justification referencing the fetched content

#### Scenario: LLM degraded — heuristic why preserved
- **WHEN** the LLM adapter is in degraded mode
- **THEN** `CandidateComponent.why` and `CandidateFile.reason` retain the heuristic string generated by the deterministic mapper
